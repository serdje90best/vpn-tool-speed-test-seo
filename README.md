## FAQ

**1) What should I test first: closest server or a specific region?**  
Start closest for a baseline, then test the region you actually need (streaming/work/geo apps). Compare both.

**2) Why do my numbers swing between runs?**  
Congestion and route changes. Run 3 tests per region, same time window, and use the median.

**3) Does AES-256 encryption make the VPN slower?**  
It adds overhead, but modern systems handle it well. Server load and routing usually matter more.

**4) Should Kill Switch stay on during testing?**  
Yes. Speed is pointless if the tunnel drops and traffic leaks. Keep it on.
